# AiYo v0.3.7

修复数据存储位置切换不生效的两个关键问题。

## Changes

- **修复启动路径标识符不匹配**: `main.rs` 中查找 `app-mode.json` 的路径硬编码了旧标识符 `com.claude-code-haha.desktop`，而 `set_app_mode` 写入 `app.path().app_config_dir()` 解析为 `com.aiyo.desktop`，导致重启后配置无法被识别
- **修复 relaunch() 环境变量继承**: 从便携模式切回系统默认模式时，`relaunch()` 会继承旧进程的 `CLAUDE_CONFIG_DIR`，导致新进程仍使用旧目录。新增 `CC_HAHA_APP_PORTABLE_DIR` 标记检查 + `else` 分支主动清除继承变量
- **统一 bundle identifier**: 将 `build-macos-arm64.sh`、`conversationService` 及测试文件中的旧标识符同步更新为 `com.aiyo.desktop`

## Notes

- 切换便携模式 ↔ 系统默认目录需要重启应用才能生效
- 切换目录不会迁移已有数据
- 外部设置 `CLAUDE_CONFIG_DIR` 的优先级保持不变
