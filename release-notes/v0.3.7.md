# AiYo v0.3.7

修复 rebrand 遗留标识符不匹配导致数据存储位置切换不生效。

## Changes

- **修复数据存储位置切换不生效**: `main.rs` 中启动时查找 `app-mode.json` 的路径硬编码了旧标识符 `com.claude-code-haha.desktop`，而 `set_app_mode` 写入的新标识是 `com.aiyo.desktop`，导致重启后配置无法被识别，存储位置切换永远不生效
- **统一 bundle identifier**: 将 `build-macos-arm64.sh`、`conversationService` 及测试文件中的旧标识符同步更新为 `com.aiyo.desktop`

## Notes

- 修复后切换便携模式 ↔ 系统默认目录需要重启应用才能生效
- 切换目录不会迁移已有数据
